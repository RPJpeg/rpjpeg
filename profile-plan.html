PROJECT OUTLINE

TEST

For this project, I really wanted to focus on the navigation. I emulated an extremely large website that has many pages and a lot of content. These projects can be very challenging using a more traditional navigation bar. It can get very complicated and you risk the user getting lost. A mega menu is a fantastic option. You get to keep the simple look of the standard navigation bar, or in this case hamburger menu, while being able to provide the user with the control to expand the navigation at will. If it becomes too overwhelming, they can simply hide the menu and move on.

Before I started coding the project, I first broke it up into divs and classes using Adobe Illustrator. I always make a site plan like this for myself to use as a reference when I start to build the HTML and CSS. I am definitely a visual person, hence the color coding, and this makes it easier for me to see how I'm going to approach each section. My biggest challenge on this case was the media card grid, and getting it to be responsive for all screen sizes. It turned out well in the end. The great thing about using classes for these cards, is that once get the class to respond well to your goal, they all just fall into place.

Another goal I had when I started this project was to setup some complex CSS classes that could handle a lot of content in the form of media cards. This method takes a lot of information and divides it up into easy-to-read cards. This makes it easy for the user to find what they are interested in and go from there. Images help break up the redundancy that you tend to get when dealing with a lot of content. The cards can easily be set up as links for a user-friendly way to take someone where you want them to go. They can also be easily swapped out for future updates.
